    User Journey & System Mechanism

    Bagaimana sistem SDLS bekerja dari awal hingga transaksi selesai

    1. Authentication & Activation

    User login menggunakan email dan diverifikasi melalui WhatsApp OTP untuk memastikan keamanan dan validitas identitas.

    Setelah login, user mengunggah skill beserta portofolio sebagai bukti kemampuan.

    2. Initial Skill-Coin

    Setelah sistem memverifikasi skill dan portofolio, user otomatis mendapatkan 1 Skill-Coin sebagai modal awal eksplorasi platform.

    3. Point Accumulation (Max 3)

    • 1 Point: Upload skill pertama
    • 2–3 Point: Transaksi dari user lain
    • Saldo maksimum: 3 Skill-Coin

    Transaksi ke-4 dan seterusnya dialihkan ke System Liquidity Reserve.

    4. Exploration & Transaction

    User mencari skill yang dibutuhkan, memilih penyedia, dan melakukan transaksi menggunakan Skill-Coin.

    Sistem menahan poin sebagai escrow hingga pekerjaan selesai.

    5. Rating & Recommendation

    Setelah pekerjaan selesai, user memberikan rating dan ulasan. Akun dengan performa tinggi masuk ke daftar Recommended Developers.

    Simplified Flow

    Login → WhatsApp OTP → Upload Skill → Claim 1 Skill-Coin → Browse Skill → Escrow Transaction → Completion → Rating → Balance Update
